Run your own

gh-widgets is one Cloudflare Worker plus a folder of static files. There is no database, no build step, and no secret to configure. Fork it, deploy it, and your READMEs use a service only you control. It fits easily in Cloudflare's free tier.

1. Fork and clone

Fork hdprajwal/gh-widgets on GitHub, then clone your fork: 

git clone https://github.com/<you>/gh-widgets
cd gh-widgets

2. Run it locally

The only tool you need is wrangler, Cloudflare's command-line tool for Workers. You can run it through npx without installing anything: 

npx wrangler dev

Open http://localhost:8787/ for the landing page, or call the API directly: http://localhost:8787/header/graph.svg?title=hello. Wrangler reloads on its own when you save a file.

3. Deploy to Cloudflare

You need a free Cloudflare account. Before deploying, open wrangler.jsonc and remove the routes block. It points the worker at this site's domain, which your account does not own: 

// wrangler.jsonc: delete these lines in your fork
"routes": [
  { "pattern": "gh-widgets.hdprajwal.dev", "custom_domain": true }
],

Then log in and deploy:

npx wrangler login
npx wrangler deploy

The first deploy prints your URL: https://gh-widgets.<your-subdomain>.workers.dev. That is it. The landing page, the builder, and the SVG API are all live.

4. Optional: custom domain

If you have a domain on Cloudflare, add the routes block back with your own hostname and deploy again. Cloudflare creates the DNS record and the certificate for you: 

"routes": [
  { "pattern": "widgets.example.com", "custom_domain": true }
],

5. Use it in a README

Open your deployed landing page, set up a header in the builder, and copy the <picture> snippet it generates. GitHub shows the dark or light version to match each visitor's theme.

Good to know

  • Cost. The free Workers plan includes 100,000 requests a day, and GitHub also caches README images on its own proxy. A personal instance will not get near the limit.
  • Self-contained. The worker calls no other service, with one exception. The logo parameter fetches an icon from simple-icons through jsDelivr and caches it for 30 days. If you skip logo, nothing external is ever called.
  • Adding widgets. Write a render function in worker.js, register its route, and describe it in public/widgets.js. The landing page builds its tabs, examples, builder, and API docs from that file.
  • No tracking. No analytics, no cookies, and no logs of your README traffic beyond Cloudflare's standard request metrics.
